1. Vision & Positionnement du Projet

TradePulse est une plateforme web SaaS de journal de trading conçue pour les traders indépendants souhaitant améliorer leur performance grâce à une analyse data-driven de leurs activités. Elle remplace les suivis manuels (Excel, Word) par un système intelligent intégré.

1.1 Problèmes résolus
•	Suivi manuel fastidieux et source d’erreurs
•	Absence de statistiques automatisées et temps réel
•	Impossibilité de visualiser l’évolution du capital et du drawdown
•	Manque d’outils d’identification des patterns gagnants / perdants
•	Aucune dimension psychologique intégrée au suivi

1.2 Proposition de valeur

🎯 Valeur ajoutée clé
→ Analyse statistique automatique de chaque trade enregistré
→ Identification des patterns rentables par stratégie, paire, session et heure
→ Journal psychologique pour comprendre et corriger ses biais émotionnels
→ Suivi multi-comptes avec objectifs personnalisés
→ Tableaux de bord visuels en temps réel

2. Fonctionnalités Principales

2.1 Gestion des utilisateurs
•	Inscription / Connexion sécurisée (email + mot de passe)
•	Authentification JWT avec refresh token
•	Récupération de mot de passe par email
•	Profil personnalisable (avatar, fuseau horaire, devise de référence)
•	Gestion des préférences d’affichage

2.2 Gestion multi-comptes de trading
Un utilisateur peut gérer plusieurs comptes de trading de types différents :

Type de compte	Exemples	Spécificités
Compte personnel	Compte chez un broker	Pas de règles imposées
Compte Prop Firm	FTMO, MyForexFunds	Règles drawdown strictes
Compte Challenge	Phase 1 & 2 FTMO	Objectifs de profit définis
Compte Virtuel	Compte papier trading	Simulation sans risque réel

Informations d’un compte
•	Nom du compte, broker, devise
•	Solde initial, solde actuel, objectif cible
•	Drawdown maximum autorisé (pour Prop Firms)
•	Date de création, statut (Actif / Inactif / Passé / Échoué)
•	Règles personnalisées (max lots, max trades/jour, jours de trading)

2.3 Module d’enregistrement des trades

Champ	Type	Description	Requis

Paire / Instrument	Select	Ex. EURUSD, XAUUSD, NAS100	Oui

2.4 Journal psychologique du trader
Chaque trade peut être enrichi d’une entrée psychologique pour identifier les biais cognitifs et améliorer la discipline :

•	État émotionnel avant le trade (slider 1-10 : calme → stressé)
•	État de confiance dans le setup (faible / moyen / élevé)
•	Respect du plan de trading (Oui / Non / Partiel)
•	Erreur identifiée (FOMO, revenge trading, trop tôt sorti...)
•	Leçon apprise
•	Humeur générale de la session

💡 Fonctionnalité avancée : Score Psychologique
Un score de discipline est calculé automatiquement sur la semaine et le mois,
basé sur le respect du plan, l’absence de revenge trading et la gestion émotionnelle.
Ce score est affiché sur le dashboard principal.

2.5 Gestion des objectifs & suivi des règles Prop Firm
•	Définition d’un objectif de profit par compte (ex. +10% en 30 jours)
•	Drawdown daily maximum (ex. -5% / jour)
•	Drawdown global maximum (ex. -10% du capital)
•	Alertes automatiques quand on approche des limites
•	Barre de progression visuelle vers l’objectif
•	Tableau de bord dédié Prop Firm avec indicateurs de conformité

4.4 Métriques de risk management
•	Drawdown maximum historique
•	Drawdown actuel (depuis dernier pic)
•	Ratio Calmar (return annuel / max drawdown)
•	Ratio de Sharpe (performance ajustée au risque)
•	Consecutive losses maximum
•	Average holding time (durée moyenne d’un trade)
•	Récupération moyenne après drawdown

4.5 Filtres avancés
Les trades peuvent être filtrés et analysés selon :
•	Période (date de début / fin)
•	Paire ou groupe de paires
•	Stratégie / Setup
•	Session de trading
•	Issue (TP / SL / BE / Manuel)
•	Type d’ordre (Buy / Sell)
•	Résultat (gagnant / perdant / breakeven)
•	État psychologique

5. Architecture Technique

5.1 Stack technologique

Couche	Technologie	Justification
Frontend	Next.js 14 + React 18	SSR, App Router, performances optimales
Styling	TailwindCSS + shadcn/ui	Design system cohérent et rapide
Graphiques	Recharts + Tremor	Composants data viz adaptés SaaS
Backend	API Routes Next.js	Serverless, intégré au frontend
Base de données	MongoDB Atlas	Flexible pour structures de trades
ORM	Mongoose	Schémas validés + index optimisés
Auth	NextAuth.js + JWT	Sessions sécurisées, OAuth possible
Upload fichiers	Cloudinary ou UploadThing	Stockage screenshots de trades
Email	Resend ou Nodemailer	Notifications et récupération MDP
Déploiement	Vercel	Préféré pour Next.js, auto-scaling

5.3 Modèles de données MongoDB

Collection users
Schéma User
{ _id, name, email, passwordHash, avatar, timezone, currency,
  preferences: { theme, dateFormat, notifications }, createdAt, updatedAt }

Collection accounts
Schéma Account
{ _id, userId, name, broker, type, currency,
  initialBalance, currentBalance, targetBalance,
  maxDailyDrawdown, maxTotalDrawdown,
  status: 'active'|'inactive'|'passed'|'failed',
  rules: { maxLots, maxTradesPerDay, tradingDays[] },
  createdAt, updatedAt }

Collection trades
Schéma Trade
{ _id, accountId, userId,
  date, pair, orderType, lot, session,
  setup, strategy, rpt, rrRatio,
  issue, resultPercent, resultDollar,
  screenshots: [url],
  psychology: { emotionalState, confidence, planFollowed,
    error, lesson, mood },
  observation, tags: [],
  createdAt, updatedAt }

Collection strategies
Schéma Strategy
{ _id, userId, name, description, rules: [],
  pairs: [], timeframes: [], active: bool, createdAt }

5.4 Index MongoDB optimisés
•	{ accountId: 1, date: -1 } — listing des trades
•	{ userId: 1, date: -1 } — dashboard global
•	{ accountId: 1, pair: 1 } — analyse par paire
•	{ accountId: 1, strategy: 1 } — analyse par stratégie
•	{ accountId: 1, resultDollar: 1 } — classements
